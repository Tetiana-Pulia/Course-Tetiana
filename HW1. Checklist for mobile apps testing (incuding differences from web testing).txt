Тестирование совместимости используется, чтобы убедиться, что ваше приложение совместимо с другими версиями ОС, различными оболочками и сторонними сервисами, а также аппаратным обеспечением устройства.
•	Корректное отображение геолокации
•	Информации об операциях (чеки и т.д.)
•	Различные способы оплаты (Google Pay, Apple Pay)
•	Тестирование датчиков (освещенности, температуры устройства, гироскоп и т.д.)
•	Тестирование прерываний работы приложения (входящий звонок/смс/push/будильник и т.д.): В какой режим переходит приложение: не закрывается ли полностью, не прекращает ли свою работу


•	Работа приложения в разных режимах («Не беспокоить»/ «В полете (авиарежим)»/ «Спящий режим»)

•	Подключение внешних устройств (карта памяти): а) Проверка работы приложения при нехватке памяти в аппарате (проверка и когда приложение установлено на память телефона, и когда на память sd карты); отдельно проверить, как будет работать, если вытянуть карту памяти, заругается ли приложение. 
Tester should check
a.       When data saved on Mobile Memory, application access them easily

c.       When data saved on SD card, application access them easily
d.      What happens when SD card is removed
e.      When data saved on cloud, application access them easily
f.        Data recovery when connection lost or SD card damaged

•	Подключение внешних устройств (наушники): а) Работа приложения при подключении проводных наушников. б) Работа приложения при подключении беспроводных наушников по Bluetooth.
•	Работа приложения в условиях низкого заряда батареи или когда телефон на зарядке.

Стрессовое тестирование Стрессовое тестирование направлено на определение эффективности производительности приложения в условиях повышенной нагрузки. Стресс-тест в этом контексте ориентирован только на мобильные устройства.
•	Высокая загрузка центрального процессора
•	Нехватка памяти
•	Загрузка батареи
•	Отказы
•	Низкая пропускная способность сети
•	Большое количество взаимодействий пользователя с приложением

Тестирование производительности Если пользователь устанавливает приложение, и оно не отображается достаточно быстро (например, в течение трех секунд), оно может быть удалено в пользу другого приложения. Аспекты потребления времени и ресурсов являются важными факторами успеха для приложения, и для измерения этих аспектов проводится тестирование производительности. Проверяем
•	Время загрузки (launch) приложения 
•	Обработка запросов
•	Кэширование данных
•	Потребление ресурсов приложением (например расход заряда батареи)
•	Application performance at times of peak load conditions and continual key pad entry
•	Application performance in low device battery and charging conditions
•	Application install/uninstall successfully within the desired timeframe
•	Displaying error messages and exits gracefully at times of low memory issues
•	App Performance at the time of a Network problem and error alert prompts
